# Demo Script

## Seven-Minute Flow

### 0:00 - Problem

A buyer sees many similar products and has to guess which seller, size, offer, and payment mode to trust. Sarthi is built to reduce that guesswork before checkout.

Close the setup with:

```text
Sarthi optimizes for kept orders, not just placed orders.
```

### 0:45 - Buyer Feed

Sign in as `asha.buyer`.

Show:

- normal marketplace feed;
- product photos, rating, delivery, seller, and trust state;
- Sarthi entry point on eligible listings.

### 1:20 - Confusion Resolver

Open a product cluster and run comparison.

Show:

- recommended listing;
- alternative listing;
- top reasons;
- uncertainty and fact-backed ranking.

Say:

```text
The cheapest listing is not automatically chosen. Sarthi checks fit, returns, proof coverage, seller state, offer truth, and uncertainty.
```

### 2:05 - Product Proof

Open product detail.

Show:

- SKU Trust Passport;
- Size Oracle;
- Galti Mat Dohrao;
- review evidence;
- source freshness;
- audit drawer.

Say:

```text
When evidence is weak, Sarthi pauses or asks for proof instead of pretending to know.
```

### 3:00 - Sarthi Samvaad

Ask:

```text
In teen mein best kaunsa hai? Mera usual L hai, kapda thin nahi chahiye.
```

Show:

- detected compare, fit, and fabric intent;
- grounded answer;
- tools/fact IDs in audit;
- Gemini or deterministic fallback provider state if visible.

### 3:50 - Checkout Confidence

Continue to checkout.

Show:

- Offer Sach Check;
- cart confidence;
- prepaid/COD guidance;
- expectation contract.

Example line:

```text
No need to rush. This price has been active for multiple days.
```

Avoid saying:

```text
This seller is fake.
```

### 4:45 - Outcome Learning

Place or simulate an order and submit feedback.

Show:

- kept/returned structured outcome;
- fit memory update when allowed;
- privacy-safe future guidance.

### 5:25 - Seller Evidence Loop

Sign in as `seller.a`.

Show:

- verification state;
- evidence coach;
- proof tasks;
- listing drafts;
- aggregate-only seller data;
- proof upload or draft submit flow.

Say:

```text
The seller gets tasks they can act on, but not private buyer memory.
```

### 6:10 - Admin Review

Sign in as `reviewer.admin`.

Show:

- seller dossiers;
- active queue;
- prescreened applications, documents, proofs, and drafts;
- risk, SLA, buyer impact, source checks;
- approve/reject/revision action;
- audit events.

Say:

```text
The AI reduces reading and routing work, but the final seller or listing decision stays human-in-loop.
```

### 6:50 - Close

End with:

```text
Sarthi connects buyer confidence, seller improvement, and admin review into one trust loop.
```

## Backup If Time Is Short

Show only:

1. Buyer comparison.
2. Product proof and checkout offer truth.
3. Seller evidence task.
4. Admin review queue.

That still demonstrates the full end-to-end loop.
